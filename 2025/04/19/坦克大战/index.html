坦克大战

前言

这些天抽空写了个坦克大战项目，写这个项目的目的主要是巩固学过的知识，以及培养做项目的思维(第一次做，很多时候不知道某个方法如何运用到这个项目中，写到400多行的时候，代码特别多，如何把方法插入其中也需要思考很久)。当然，现在并没有特别完善，只是个半成品，但是，当你快完成项目的雏形时，更多是有一种成就感，和不断写下去的动力

  • 需要的知识

    • java绘图（java swing包的使用）

    • 多线程

    • IO流

    • java基本语法

游戏基本搭建

绘图

java绘图时候，(x,y)指代的是整个物体左上角的那个点，也就是说，整个物体的是基于坐上角那个点坐标进行变更

java绘图技术

  • paint(Graphics g) : 绘制组件外观
  • repaint() : 刷新组件外观（这对于运行十分重要，必须要刷新才能有那种动态感）

我们可以基于此，来绘画出坦克的外貌

这时候，我们坦克的外形便生成了

同样，画子弹也是如此

事件处理机制

java事件处理采用”委派事件模型”，详细可见Java委托事件模型解析-CSDN博客，简单来说就是产生事件的对象把事件交给信息的监听者来处理

坦克大战中，目前我们需要处理的事件有以下几个：

  • 按下键盘 “右侧的上下左右” 来控制坦克进行上下左右的移动
  • 按下J键，发射子弹
  • 点击右上角的”x”时，程序结束并且最小化
  • 点击开始页面的player , 游戏进入开始状态

游戏拆分

Tank类

主要定义tank相关属性如下，是Enemy和Hero的父类

属性 含义 方法 用途
int x int y 坦克坐标 moveUp 向上移动
int direct 坦克方向 moveDown 向下移动
boolean isLive 坦克是否存活 moveLeft 向左移动

Hero类

Hero是玩家能够操作的坦克 ，是Tank类的子类

属性 方法 用途
继承Tank shotEnemyTank 射击敌方坦克，用于点击射出的子弹

Enemy类

该类是敌方坦克，继承Tank，和Thread（线程）

属性 方法 用途
继承Tank run 重写线程的run方法，进行随机移动
isTouchEnemyTank 判断敌方坦克是否重叠

这里最主要的是集合Vector的使用，能够确保多线程的安全性，让敌方坦克互相不重叠

Bomb类

炸弹类，用于产生爆炸效果

属性 含义 方法 用途
int x int y 炸弹坐标 lifeDown 控制炸弹生命
boolean isLive 炸弹是否存活
int life 炸弹生命值

Recorder类

该类用来记录玩家击杀敌方坦克的数量，并保存到文件中，主要用到IO流

Shot类

shot类定义了子弹的相关，属性是一个线程，继承Thread类

MyPanel类

这个类是整个游戏绘图区域，在这里进行游戏的绘制，实现游戏效果

实现键盘监听，Runnable,鼠标监听的接口

paint方法用来进行绘画

非常重要的run方法，用来刷新页面，实现动态感

还有很多方法，在这里就不多赘述，等完善了一并贴出

GameFrame类

在这里启动游戏

最后效果

每当击杀一名敌人后，右边的数量加1

如果自己被敌方坦克击中，就会弹出GameOver

如果把全部坦克击杀，则显示获胜

总结

还有很多不完善的地方，比如坦克和障碍物的碰撞机制，以及一些小bug，等基本完善后这个项目也就算成功了
